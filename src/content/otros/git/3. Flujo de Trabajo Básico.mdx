### 3. **Flujo de Trabajo Básico en Git**

El flujo de trabajo básico en Git implica los pasos fundamentales para crear, modificar y gestionar un proyecto, lo que te permitirá realizar un seguimiento de los cambios en los archivos de tu repositorio. En este tema, cubriremos las operaciones más comunes de Git que forman parte del ciclo de trabajo diario de un desarrollador.

#### **1. Crear un nuevo repositorio con `git init`**

El primer paso para comenzar a usar Git en un proyecto es crear un repositorio. Un repositorio es un espacio donde Git almacena todos los archivos y el historial de cambios.

- **Comando para crear un repositorio:**
  Para crear un repositorio Git en una carpeta local, navega a esa carpeta en la terminal y ejecuta:

  ```bash
  git init
  ```

  Esto crea un nuevo repositorio vacío en esa carpeta, añadiendo un subdirectorio oculto llamado `.git`, que contiene todos los archivos y configuraciones necesarios para gestionar el control de versiones.

- **Verificación del repositorio:**
  Una vez que hayas ejecutado `git init`, puedes verificar que el repositorio ha sido creado correctamente ejecutando:

  ```bash
  git status
  ```

  Si el repositorio está vacío, Git te indicará que no hay archivos que rastrear.

#### **2. Clonar un repositorio existente con `git clone`**

Si deseas empezar a trabajar en un proyecto que ya existe en otro lugar, puedes clonar un repositorio remoto a tu máquina local. El comando `git clone` descarga una copia completa del repositorio, incluidos todos los archivos y el historial de cambios.

- **Comando para clonar un repositorio:**
  Para clonar un repositorio, ejecuta el siguiente comando, sustituyendo la URL por la del repositorio que deseas clonar:

  ```bash
  git clone https://github.com/usuario/repositorio.git
  ```

  Esto creará una carpeta con el nombre del repositorio y descargará todos los archivos del repositorio en ella.

- **Verificación del repositorio clonado:**
  Después de clonar el repositorio, puedes navegar a la carpeta del repositorio y usar `git status` para verificar que Git está rastreando todos los archivos.

#### **3. Agregar archivos con `git add`**

Una vez que hayas creado o clonado un repositorio, puedes comenzar a trabajar en los archivos del proyecto. Después de realizar cambios, necesitarás agregar esos archivos al **área de preparación** (staging area) para que estén listos para ser confirmados (commit).

- **Comando para agregar archivos:**
  Para agregar un archivo específico, usa:

  ```bash
  git add nombre_del_archivo
  ```

  Para agregar todos los archivos modificados en el directorio actual, usa el siguiente comando:

  ```bash
  git add .
  ```

  Este comando prepara los cambios en los archivos para que estén listos para ser confirmados. Sin embargo, ten en cuenta que esto no guarda los cambios en el historial, solo los prepara.

#### **4. Realizar un commit con `git commit`**

Después de agregar los archivos al área de preparación, el siguiente paso es hacer un commit. Un commit es una instantánea de tus cambios en el proyecto, y Git guarda información sobre qué archivos se modificaron, quién los modificó y cuándo se realizaron los cambios.

- **Comando para hacer un commit:**
  Para realizar un commit, usa:

  ```bash
  git commit -m "Mensaje del commit"
  ```

  El mensaje de commit debe describir de manera clara y concisa los cambios realizados en el código, para que otros (y tú mismo en el futuro) puedan entender lo que se hizo.

- **Verificación de los commits:**
  Puedes ver el historial de commits usando:

  ```bash
  git log
  ```

  Este comando muestra todos los commits realizados en el repositorio, con información como el identificador del commit, el autor y el mensaje de commit.

#### **5. Ver el estado del repositorio con `git status`**

Es importante verificar el estado de tu repositorio regularmente. El comando `git status` te indica cuáles archivos han sido modificados, cuáles están listos para ser confirmados y si hay archivos que no están siendo rastreados por Git.

- **Comando para verificar el estado del repositorio:**

  ```bash
  git status
  ```

  La salida de este comando te mostrará si hay archivos sin agregar, archivos modificados o archivos preparados para el commit.

#### **6. Visualizar cambios con `git diff`**

Si deseas ver exactamente qué cambios has realizado en los archivos desde el último commit, puedes usar el comando `git diff`. Este comando compara las versiones de los archivos y muestra las diferencias línea por línea.

- **Comando para ver las diferencias:**
  
  ```bash
  git diff
  ```

  Este comando muestra las diferencias entre los archivos modificados y la última versión confirmada (commit).

#### **7. Sincronización con repositorios remotos (Push y Pull)**

Una vez que hayas realizado un commit, es posible que desees compartir tus cambios con otros colaboradores o obtener los cambios más recientes desde un repositorio remoto. Para ello, se utilizan los comandos `git push` y `git pull`.

- **Subir cambios al repositorio remoto con `git push`:**
  
  Para subir tus commits al repositorio remoto (por ejemplo, en GitHub, GitLab o Bitbucket), usa:

  ```bash
  git push origin nombre_de_la_rama
  ```

  Aquí, `origin` es el nombre del repositorio remoto (por defecto, Git asigna este nombre al repositorio remoto clonado), y `nombre_de_la_rama` es la rama en la que estás trabajando (por ejemplo, `main` o `master`).

- **Obtener cambios desde el repositorio remoto con `git pull`:**
  
  Para actualizar tu repositorio local con los cambios más recientes desde el repositorio remoto, usa:

  ```bash
  git pull origin nombre_de_la_rama
  ```

  Este comando descarga los cambios más recientes y fusiona esos cambios con tu rama local.

#### **8. Crear y cambiar de ramas con `git branch` y `git checkout`**

El uso de ramas es uno de los puntos fuertes de Git. Las ramas te permiten trabajar en nuevas características o experimentar sin afectar la rama principal del proyecto. Existen dos operaciones principales para trabajar con ramas:

- **Comando para crear una nueva rama:**

  ```bash
  git branch nombre_de_la_rama
  ```

- **Comando para cambiar a otra rama:**

  ```bash
  git checkout nombre_de_la_rama
  ```

Si deseas crear y cambiar a una nueva rama en un solo paso, puedes usar el siguiente comando:

```bash
git checkout -b nombre_de_la_rama
```

- **Ver las ramas existentes:**

  Para listar las ramas locales en tu repositorio, usa:

  ```bash
  git branch
  ```

#### **9. Fusionar ramas con `git merge`**

Una vez que hayas realizado cambios en una rama (por ejemplo, en una nueva característica o corrección de errores), querrás integrar esos cambios a la rama principal (generalmente llamada `main` o `master`). Esto se hace mediante el comando `git merge`.

- **Comando para fusionar una rama:**
  
  Primero, asegúrate de estar en la rama a la que deseas fusionar los cambios (por ejemplo, `main`), luego usa:

  ```bash
  git merge nombre_de_la_rama
  ```

  Git intentará fusionar los cambios de la rama especificada con la rama actual. Si hay conflictos, Git te avisará y deberás resolverlos manualmente.


1. **Crear o clonar un repositorio**: `git init` / `git clone`.
2. **Agregar archivos modificados**: `git add nombre_del_archivo`.
3. **Confirmar cambios (commit)**: `git commit -m "Mensaje del commit"`.
4. **Verificar estado**: `git status`.
5. **Subir cambios al repositorio remoto**: `git push origin nombre_de_la_rama`.
6. **Obtener cambios del repositorio remoto**: `git pull origin nombre_de_la_rama`.
7. **Crear y cambiar de ramas**: `git branch nombre_de_la_rama` / `git checkout nombre_de_la_rama`.
8. **Fusionar ramas**: `git merge nombre_de_la_rama`.

>Este flujo básico es el corazón del trabajo diario con Git y cubre las acciones fundamentales que utilizarás a medida que desarrollas tu proyecto.
