### 4. **Operaciones Básicas en Git**

Las operaciones básicas en Git son esenciales para gestionar el control de versiones de tu proyecto. Estas operaciones te permiten crear, modificar, revisar y gestionar archivos y el historial de tu repositorio. En este tema, se desarrollarán las acciones más comunes y útiles para trabajar con Git de manera efectiva.

---

#### **1. Agregar Archivos con `git add`**

Antes de hacer un commit, necesitas agregar los archivos que han sido modificados al área de preparación (staging area). Esto indica a Git qué cambios deben ser guardados en el próximo commit.

- **Comando para agregar un archivo específico:**

  Si solo quieres agregar un archivo específico a la zona de preparación, utiliza el siguiente comando:

  ```bash
  git add nombre_del_archivo
  ```

  Esto añadirá el archivo especificado al área de preparación.

- **Comando para agregar todos los archivos modificados:**

  Si has modificado varios archivos y deseas agregar todos esos archivos al área de preparación, usa:

  ```bash
  git add .
  ```

  El punto (`.`) indica que todos los archivos modificados deben ser agregados. Esto es útil cuando has trabajado en varios archivos y quieres preparar todos esos cambios a la vez para el commit.

- **Comando para agregar archivos específicos con patrones:**

  Si solo deseas agregar ciertos tipos de archivos, como todos los archivos `.txt`, puedes usar un patrón:

  ```bash
  git add *.txt
  ```

---

#### **2. Realizar un Commit con `git commit`**

Un commit es una "instantánea" de los archivos en tu proyecto en un momento dado. Una vez que los archivos han sido agregados al área de preparación con `git add`, puedes hacer un commit para guardar esos cambios en el historial del repositorio.

- **Comando básico para hacer un commit:**

  ```bash
  git commit -m "Mensaje del commit"
  ```

  - El parámetro `-m` se utiliza para agregar un mensaje de commit, el cual debe ser breve pero informativo.
  - Ejemplo de mensaje de commit: `git commit -m "Corregir error en la validación de usuario"`

- **Comando para realizar un commit de todos los cambios (incluyendo nuevos archivos no rastreados):**

  Si quieres realizar un commit que incluya todos los cambios, incluidos archivos nuevos, puedes usar:

  ```bash
  git commit -am "Mensaje del commit"
  ```

  **Nota:** La opción `-a` sólo funciona con archivos que ya están siendo rastreados por Git (es decir, no nuevos archivos). Si agregas un archivo nuevo, primero tendrás que usar `git add`.

- **Verificar commits realizados con `git log`:**

  Después de hacer un commit, puedes ver el historial de commits realizados en el repositorio con:

  ```bash
  git log
  ```

  Este comando te mostrará una lista de los commits realizados, con detalles como el identificador (hash) de cada commit, el autor, la fecha y el mensaje del commit.

---

#### **3. Verificar el Estado del Repositorio con `git status`**

El comando `git status` es muy útil para ver el estado actual de tu repositorio. Te permite conocer qué archivos están siendo rastreados, cuáles han sido modificados y cuáles están listos para ser confirmados (committed).

- **Comando para ver el estado del repositorio:**

  ```bash
  git status
  ```

  La salida de `git status` te dirá si hay archivos modificados, si hay archivos sin agregar, y si el área de preparación está lista para ser confirmada.

  Ejemplo de salida:

  ```bash
  On branch main
  Changes to be committed:
    (use "git reset HEAD <file>..." to unstage)
          modified:   archivo1.txt
          new file:   archivo2.txt
  ```

  Si ves algo como "Changes not staged for commit", significa que tienes archivos modificados que no han sido agregados al área de preparación.

---

#### **4. Visualizar Cambios con `git diff`**

El comando `git diff` te permite ver las diferencias entre el estado actual de los archivos y la última versión confirmada. Esto es útil para revisar qué cambios has hecho antes de realizar un commit.

- **Comando básico para ver las diferencias:**

  ```bash
  git diff
  ```

  Esto mostrará las diferencias línea por línea de los archivos modificados.

- **Comando para ver las diferencias de un archivo específico:**

  Si solo deseas ver las diferencias de un archivo específico, puedes hacerlo con:

  ```bash
  git diff nombre_del_archivo
  ```

  Ejemplo:

  ```bash
  git diff archivo1.txt
  ```

- **Ver diferencias entre commits:**

  Si deseas comparar dos commits específicos, puedes usar:

  ```bash
  git diff commit_id1 commit_id2
  ```

  Esto te permitirá ver qué cambios se han realizado entre dos puntos específicos en el historial del proyecto.

---

#### **5. Ver el Historial de Commits con `git log`**

El comando `git log` te muestra el historial de los commits realizados en tu repositorio, lo que te permite ver cómo ha evolucionado el proyecto.

- **Comando básico para ver el historial de commits:**

  ```bash
  git log
  ```

  La salida incluye detalles como el hash del commit, la fecha, el autor y el mensaje del commit. Esto es útil para hacer un seguimiento de qué cambios se han realizado en el repositorio y cuándo.

- **Opciones útiles con `git log`:**

  - **Limitar el número de commits:**

    Si solo deseas ver un número específico de commits, puedes usar la opción `-n`:

    ```bash
    git log -n 5
    ```

    Esto muestra los últimos 5 commits.

  - **Mostrar un resumen más compacto:**

    Si prefieres un formato más resumido, usa:

    ```bash
    git log --oneline
    ```

    Este comando mostrará una línea por commit, que incluye el hash corto y el mensaje del commit.

  - **Ver los cambios realizados en cada commit:**

    Puedes agregar la opción `-p` para ver los detalles de los cambios de cada commit:

    ```bash
    git log -p
    ```

---

#### **6. Eliminar Archivos con `git rm`**

Si decides que un archivo ya no debe estar en el repositorio, puedes eliminarlo de Git y del sistema de archivos con `git rm`.

- **Comando para eliminar un archivo:**

  ```bash
  git rm nombre_del_archivo
  ```

  Esto eliminará el archivo tanto del repositorio como del directorio local.

- **Eliminar un archivo solo del repositorio, pero mantenerlo localmente:**

  Si solo quieres eliminar el archivo del seguimiento de Git, pero mantenerlo en tu sistema local, usa:

  ```bash
  git rm --cached nombre_del_archivo
  ```

---

#### **7. Descartar Cambios con `git checkout`**

Si has realizado cambios en archivos, pero decides que no quieres conservarlos, puedes descartarlos y restaurar los archivos al último commit confirmado.

- **Comando para descartar cambios no confirmados:**

  ```bash
  git checkout -- nombre_del_archivo
  ```

  Esto descarta los cambios realizados en el archivo y lo restaura a la versión más reciente confirmada.

- **Descartar todos los cambios en los archivos:**

  Si deseas descartar todos los cambios no confirmados en los archivos del repositorio, puedes usar:

  ```bash
  git checkout -- .
  ```

1. **Agregar archivos**: `git add [archivo]` o `git add .` para agregar todos los archivos modificados.
2. **Realizar un commit**: `git commit -m "Mensaje del commit"`.
3. **Ver el estado del repositorio**: `git status`.
4. **Ver diferencias de los cambios**: `git diff`.
5. **Ver el historial de commits**: `git log`.
6. **Eliminar archivos**: `git rm [archivo]`.
7. **Descartar cambios no confirmados**: `git checkout -- [archivo]`.

>Estas son las operaciones fundamentales que usarás regularmente para interactuar con Git. Con estas herramientas, puedes gestionar tus archivos, mantener un historial de cambios y colaborar de manera eficiente con otros desarrolladores.
