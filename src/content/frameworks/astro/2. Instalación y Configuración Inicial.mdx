### 2. **Instalación y Configuración Inicial de Astro**

Astro es un framework moderno que facilita la creación de sitios web rápidos y optimizados. En esta sección, cubriremos los pasos para **instalar y configurar Astro**, desde la instalación básica hasta la configuración inicial del proyecto. Esto te permitirá comenzar a trabajar en tu primer proyecto con Astro de manera eficiente.

---

#### **Requisitos Previos**

Antes de comenzar con la instalación de Astro, asegúrate de tener los siguientes **requisitos previos**:

- **Node.js**: Astro es un framework que depende de Node.js, por lo que necesitarás tenerlo instalado en tu máquina.
  - Puedes verificar si ya tienes Node.js instalado ejecutando el siguiente comando en tu terminal:
    ```bash
    node -v
    ```
  - Si no tienes Node.js, puedes descargarlo desde su [página oficial](https://nodejs.org/).
  
- **npm** o **Yarn**: Son los administradores de paquetes que se utilizan en Astro. **npm** se instala junto con Node.js, pero también puedes usar **Yarn** si prefieres. Ambos sirven para gestionar dependencias y ejecutar comandos de desarrollo.

---

#### **Paso 1: Crear un Nuevo Proyecto con Astro**

Para comenzar a trabajar con Astro, el primer paso es crear un nuevo proyecto. Puedes hacerlo fácilmente usando el **creador de proyectos** oficial de Astro, que te permite configurar un nuevo proyecto en cuestión de minutos.

1. **Usando npm (opción recomendada)**:
   - Abre tu terminal y ejecuta el siguiente comando para crear un proyecto nuevo con Astro:
     ```bash
     npm create astro@latest
     ```
   - Este comando descargará y ejecutará el **creador de proyectos de Astro**. Se te pedirá que elijas un nombre para tu proyecto y, si lo deseas, una plantilla predeterminada.

2. **Usando Yarn**:
   Si prefieres usar Yarn en lugar de npm, puedes crear un nuevo proyecto con:
   ```bash
   yarn create astro
   ```

   Esto hará lo mismo que el comando anterior, pero utilizando Yarn como administrador de paquetes.

3. **Seleccionar una Plantilla**:
   Durante la creación del proyecto, Astro te ofrecerá diferentes plantillas para elegir. Algunas de las plantillas más comunes son:
   - **"Blog"**: Una plantilla sencilla para blogs estáticos.
   - **"Portfolio"**: Plantilla orientada a la creación de portafolios personales.
   - **"Basic"**: Una plantilla mínima para comenzar desde cero.
   
   Puedes elegir una plantilla predeterminada o empezar con una plantilla en blanco para tener total control sobre la estructura del proyecto.

---

#### **Paso 2: Instalación de Dependencias**

Una vez que hayas creado el proyecto, el siguiente paso es instalar todas las dependencias necesarias para que el proyecto funcione correctamente.

1. **Instalar dependencias usando npm**:
   - Dirígete al directorio del proyecto recién creado:
     ```bash
     cd nombre-de-tu-proyecto
     ```
   - Instala todas las dependencias utilizando el siguiente comando:
     ```bash
     npm install
     ```

2. **Instalar dependencias usando Yarn**:
   Si estás usando Yarn, instala las dependencias con:
   ```bash
   yarn
   ```

Astro utilizará el archivo `package.json` para gestionar las dependencias del proyecto, lo que incluye las dependencias esenciales como Astro mismo, y otras dependencias de desarrollo o complementos que puedas agregar en el futuro.

---

#### **Paso 3: Estructura Básica del Proyecto**

Una vez que la instalación haya finalizado, el proyecto tendrá una estructura de directorios y archivos básica. Esta es la estructura por defecto que puedes esperar ver:

```
/nombre-de-tu-proyecto
  ├── /public          # Archivos estáticos como imágenes y fuentes
  ├── /src             # Archivos fuente del proyecto (componentes, páginas, etc.)
  │   ├── /components  # Componentes reutilizables de Astro
  │   ├── /layouts     # Layouts de las páginas
  │   └── /pages       # Archivos de páginas (Markdown, Astro, etc.)
  ├── astro.config.mjs # Archivo de configuración de Astro
  ├── package.json     # Dependencias y scripts del proyecto
  ├── README.md        # Archivo de documentación
```

- **`/src/pages`**: Aquí es donde irán las páginas del sitio web. Las páginas pueden ser archivos `.astro`, `.md` (Markdown) o `.jsx`/`.tsx` si estás usando un framework como React o Vue.
  
- **`/src/components`**: En esta carpeta se almacenarán los componentes reutilizables que puedes importar en tus páginas o layouts. Los componentes pueden ser archivos `.astro` o cualquier archivo compatible con el framework que estés usando (React, Vue, etc.).

- **`astro.config.mjs`**: Este es el archivo de configuración principal de Astro. Aquí puedes configurar ajustes como las rutas, plugins, el comportamiento de la construcción, etc.

---

#### **Paso 4: Ejecutar el Servidor de Desarrollo**

Una vez que hayas instalado las dependencias y entendido la estructura básica del proyecto, puedes comenzar a desarrollar y ver los cambios en tiempo real.

1. **Iniciar el servidor de desarrollo**:
   - Ejecuta el siguiente comando para iniciar el servidor de desarrollo:
     ```bash
     npm run dev
     ```

2. **Iniciar con Yarn**:
   Si estás usando Yarn, ejecuta:
   ```bash
   yarn dev
   ```

Esto arrancará un servidor de desarrollo local en tu máquina y podrás ver tu proyecto en tu navegador. Por lo general, se iniciará en `http://localhost:3000/`. Cada vez que realices cambios en los archivos de tu proyecto, Astro los reflejará en tiempo real sin necesidad de reiniciar el servidor manualmente.

---

#### **Paso 5: Configuración Inicial (Opcional)**

Aunque el proyecto se crea con una configuración básica, es posible que quieras personalizar algunos aspectos, como:

1. **Configuración del archivo `astro.config.mjs`**:
   - El archivo `astro.config.mjs` es donde se encuentran las configuraciones globales de tu proyecto, como la configuración de rutas, el manejo de plugins, etc. Un archivo `astro.config.mjs` típico se ve algo así:
     ```js
     import { defineConfig } from 'astro/config';

     export default defineConfig({
       site: 'https://tusitio.com', // Define la URL de tu sitio
       markdownOptions: {
         shikiConfig: { theme: 'nord' }, // Configuración para resaltar código en Markdown
       },
     });
     ```
   - **Propiedad `site`**: Define la URL del sitio para que Astro pueda generar las URLs absolutas durante la compilación.
   - **`markdownOptions`**: Configura cómo se debe procesar el contenido de Markdown, como el resaltado de sintaxis.

2. **Instalar complementos (plugins)**:
   - Astro permite la integración de varios **plugins** para extender sus funcionalidades, como soporte para imágenes, herramientas de SEO, y más.
   - Para instalar un plugin, simplemente ejecuta el comando de instalación correspondiente y luego agrégalo a tu archivo de configuración.

---

#### **Paso 6: Construir y Desplegar el Proyecto**

Una vez que hayas realizado los cambios y estés listo para lanzar tu sitio, puedes generar la versión final y optimizada para producción.

1. **Generar el sitio para producción**:
   - Usa el siguiente comando para crear una versión optimizada y lista para desplegar:
     ```bash
     npm run build
     ```

2. **Desplegar el sitio**:
   - Una vez que el sitio esté construido, puedes desplegarlo en plataformas como **Vercel**, **Netlify** o **GitHub Pages**. Astro facilita el despliegue en estas plataformas a través de su integración directa y su salida optimizada.

---

### **Resumen del Proceso de Instalación y Configuración**

1. **Instalación de dependencias**: Usa `npm create astro@latest` o `yarn create astro` para crear un nuevo proyecto con Astro.
2. **Estructura del proyecto**: Comprende la estructura básica del proyecto que incluye carpetas para páginas, componentes y layouts.
3. **Servidor de desarrollo**: Usa `npm run dev` o `yarn dev` para iniciar el servidor local y ver tu sitio en tiempo real.
4. **Configuración**: Personaliza la configuración a través del archivo `astro.config.mjs` según sea necesario.
5. **Despliegue**: Una vez que hayas terminado, genera una versión de producción con `npm run build` y despliega el sitio en plataformas populares.

---

>Con estos pasos, habrás instalado y configurado correctamente un proyecto Astro. Ahora estarás listo para comenzar a desarrollar tu sitio web, personalizarlo y aprovechar las potentes características de Astro para obtener un sitio optimizado y de alto rendimiento.
